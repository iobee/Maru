# Maru 风格好搭子组合控制设计

日期：2026-07-12  
状态：已实现并验收  
范围：首页组合管理 Stage Manager 与 Dock 自动隐藏，不新增页面，不修改窗口规则模型

## 1. 目标

首页不再把 Stage Manager 和 Dock 自动隐藏作为两个同级选项直接展示。用户只需要理解一个推荐搭配：`Maru 风格好搭子`。

这个组合控制负责：

- 一键开启 Stage Manager
- 一键开启 Dock 自动隐藏
- 在两项都开启时一键关闭两项
- 仅有一项开启时显示 `部分开启`，再次打开组合开关只补齐缺失项
- 把两项独立开关收进默认折叠的 `单独设置`

## 2. 产品边界

包含：

- 首页新增唯一可见的推荐组合卡
- 组合状态：`推荐开启`、`部分开启`、`搭配已就绪`
- 默认折叠的 Stage Manager 与 Dock 自动隐藏独立开关
- 两项系统设置各自独立读写，允许部分成功
- 任一写入错误始终显示在组合卡中，不因折叠而隐藏

不包含：

- 自动替用户开启组合设置
- 新增导航页面、菜单栏项或快捷键
- 管理 Dock 位置、放大、尺寸等其他偏好
- 修改 Maru 的窗口选择、规则或几何算法

## 3. 首页结构与视觉层级

首页顺序固定为：

1. 页面头部
2. 窗口自动管理主控制卡
3. `Maru 风格好搭子` 推荐组合卡
4. 呼吸空间控制卡

主控制卡仍是唯一 Hero。组合卡使用常规功能表面，以稳定实体背景、轻蓝色偏和细描边表达推荐关系，不与主开关争夺视觉中心。蓝色仍是唯一主强调色。

## 4. 组合状态与操作语义

| Stage Manager | Dock 自动隐藏 | 组合状态 | 组合开关 | 用户打开后的写入 |
| --- | --- | --- | --- | --- |
| 关 | 关 | 推荐开启 | 关 | 两项都开启 |
| 开 | 关 | 部分开启 | 关 | 仅开启 Dock 自动隐藏 |
| 关 | 开 | 部分开启 | 关 | 仅开启 Stage Manager |
| 开 | 开 | 搭配已就绪 | 开 | 用户关闭时两项都关闭 |

组合写入不做事务回滚。macOS 系统设置可能独立失败；部分成功时保留真实系统状态，并显示 `部分开启` 和对应错误，让用户可以重试或展开 `单独设置` 调整。

## 5. 文案

- 标题：`Maru 风格好搭子`
- 描述：`一键开启 Stage Manager 和 Dock 自动隐藏，让窗口分组、居中与呼吸留白自然配合。`
- 关闭状态动作：`一键开启推荐搭配`
- 部分状态动作：`补齐推荐搭配`
- 开启状态动作：`推荐搭配正在使用`
- 折叠入口：`单独设置`
- 折叠说明：`需要时分别调整这两项系统功能`
- 独立项：`Stage Manager`、`自动隐藏 Dock`

## 6. 技术设计

- `StageManagerSettings` 与 `DockSettings` 继续分别拥有真实系统状态和错误状态。
- `HomeDashboardState` 负责推导组合状态、展示文案和最小写入计划。
- `CompanionChangePlan` 用可选目标值表示是否需要写入某一项，避免重复设置已经满足目标的系统偏好。
- `HomeDashboardView` 执行写入计划并渲染组合卡；独立项只在 `DisclosureGroup` 展开后出现。
- 首页出现时刷新两项真实系统状态。

## 7. 验收标准

- 首页默认只出现一个名为 `Maru 风格好搭子` 的系统搭配选项
- 主控制卡排在组合卡前，符合首页 Hero 优先级
- 一键开启能同时请求开启 Stage Manager 与 Dock 自动隐藏
- 部分开启时只补齐缺失项，不重复写入已开启项
- 两项都开启时可以一键关闭两项
- 独立设置默认折叠，展开后可分别操作
- 任一设置失败时状态与错误可见，且另一项仍可成功
- UI 符合 `docs/PRODUCT-UI-DESIGN-STANDARD.md`
- 相关单元测试、全量测试和 release build 通过

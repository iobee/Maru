# HiWindowGuy 产品级 UI 标准 Spec

> 说明：本文件是 2026-04-18 的原始产品级 UI spec 记录稿。正式生效的项目级规范已经整理为 `docs/PRODUCT-UI-DESIGN-STANDARD.md`。后续主窗口 UI 工作请优先遵循正式规范，本文仅作为历史来源与补充参考。

日期：2026-04-18  
状态：已完成风格确认，作为后续页面改造统一标准  
范围：覆盖主窗口现有页面与后续新增页面，不涉及菜单栏图标与系统权限弹窗

## 1. 目标

把 HiWindowGuy 已经确认下来的首页与左侧栏风格，固化成一套可重复执行的产品级 UI 标准，供后续“应用规则”“日志”以及新增功能页面统一使用。

这份规范不再回答“首页应该怎么改”，而是回答：

- HiWindowGuy 以后整体应该长成什么样
- 哪些区域可以使用玻璃感，哪些区域必须稳住
- 组件、层级、间距和状态表达应当遵循什么规则
- 后续页面改造时什么可以继承，什么不能再随意发挥

## 2. 已确认的产品方向

用户已经确认的总体方向如下：

- 产品气质：`macOS utility console`
- 材质策略：`restrained sidebar glass + solid dark content surfaces`
- 首页定位：`主控制面板，不是网页式 dashboard`
- 侧栏方向：`Apple Cues Only`
- 搜索框：`默认不引入`
- 主强调色：`蓝色`
- 玻璃策略：`轻度玻璃，只用于结构层`

这意味着 HiWindowGuy 的设计目标不是模仿苹果系统设置的全部结构，也不是做一套网页后台 UI，而是做一个更像原生 macOS 工具的桌面控制台。

## 3. 一句话风格定义

HiWindowGuy 的标准风格定义为：

`左侧栏有轻度原生玻璃感，右侧内容区以稳定深色实体表面为主，信息层级依赖结构、描边和留白，而不是依赖多色块、重阴影和夸张材质。`

## 4. 核心设计原则

### 4.1 结构先于装饰

- 专业感主要来自信息结构、块级关系和交互优先级
- 不靠彩色卡片、粗阴影和大面积发光制造“设计感”

### 4.2 左右分工明确

- 左侧栏负责导航、产品识别和轻度氛围
- 右侧内容区负责功能表达和操作承载
- 左右不能争抢主视觉

### 4.3 材质必须克制

- 玻璃感只允许出现在结构层和少量主控制层
- 不允许整页多个区域同时大面积使用强材质
- 默认优先稳定表面，其次才是材质气氛

### 4.4 蓝色是唯一主强调色

- 蓝色用于选中态、主开关、关键数值、主状态徽标
- 绿色、红色、黄色仅用于状态语义，不参与装饰
- 不再使用紫色、青色、绿色等多套强调色并存的旧风格

### 4.5 像桌面工具，不像网页后台

- 避免大块 CTA 感选中项
- 避免网页式统计卡阵列
- 避免满屏圆角卡片彼此竞争

## 5. 页面结构标准

所有主窗口页面采用以下结构逻辑：

### 5.1 左侧结构层

用途：

- 产品品牌识别
- 一级导航
- 底部运行状态

约束：

- 不承担页面主内容的视觉中心
- 不承载复杂业务交互
- 不承载大块统计信息

### 5.2 右侧内容层

用途：

- 页面标题与说明
- 主功能卡片
- 参数控制
- 列表、日志、规则编辑等业务内容

约束：

- 主功能卡片优先放在页面上半部
- 页面内只保留清晰的一级、二级视觉重点
- 次级信息用较低对比和较小字号处理，不抢主内容

## 6. 左侧栏标准

左侧栏是全产品唯一允许长期保留原生玻璃感的结构区。

### 6.1 左侧栏材质

- 使用原生 `NSVisualEffectView` 侧栏材质桥接
- 默认材质：`.sidebar`
- 当前采用的混合方式：`behindWindow`
- 不额外叠顶部高光
- 可以保留极轻的分隔线和轻阴影，但不能叠加厚重发光

### 6.2 左侧栏容器

- 采用贴边面板，而不是悬浮卡片
- 背景延伸到标题栏后面，让 traffic lights 处于同一块材质之内
- 右侧只保留一条轻分隔，不做厚描边容器

### 6.3 左侧栏尺寸与节奏

当前确认可作为默认 token：

- 侧栏宽度：`248`
- 顶部内容内边距：`top 20`
- 水平内边距：`14`
- 底部内边距：`14`
- 导航行高：`46`

这些数值可在个别页面按密度做轻微调整，但不能破坏整体节奏。

### 6.4 品牌区

- 保留 `Hi Window Guy + 图标`
- 品牌区是产品名，不是页面标题
- 当前品牌图标默认尺度：`38 x 38`
- 品牌区上下留白保持紧凑，不再使用横幅式大标题布局

### 6.5 导航项

- 导航项必须表现为列表中的一行，而不是网页按钮
- 当前选中项必须是明显蓝色高亮行
- 图标底板允许保留，但必须统一尺寸与圆角
- 不增加搜索、不增加账户卡、不增加系统设置式复杂分组

## 7. 右侧内容区标准

右侧内容区以稳定、克制的实体表面为主，不允许整体变成“第二块玻璃面板”。

### 7.1 页面容器

当前默认结构：

- 右侧内容区默认不设固定内容宽度
- 页面整体水平内边距：`34`
- 页面整体垂直内边距：`30`
- 顶级模块间距：`24`

右侧页面默认通过统一页边距控制留白，而不是通过固定内容宽度制造单边空白。后续页面如果因为信息密度确实需要特殊宽度约束，必须作为明确例外说明，而不是回到旧的固定承载宽度做法。

### 7.2 页面标题层级

- 页面标题：`largeTitle + bold`
- 页面副标题：`body + secondary`
- 模块标题：`title2 / title3 + semibold`

每个页面最多使用三层主要文字层级：

- 页面标题
- 模块标题
- 正文/说明

不允许继续叠加更多粗体和更大的字号制造“层级”。

### 7.3 内容表面分工

右侧内容表面分成两类：

#### A. 主控制表面

适用：

- 首页 Hero Control
- 后续页面的主开关区域
- 需要承载最高优先级交互的卡片

标准：

- 可使用轻度材质或很浅的玻璃感
- 必须配合细描边
- 必须保持克制，不能像漂浮果冻块

#### B. 常规功能表面

适用：

- 参数控制卡
- 规则编辑块
- 日志过滤与列表承载区
- 辅助信息模块

标准：

- 以深色实体面为主
- 用轻描边、轻色偏和圆角建立层次
- 不使用强玻璃

## 8. 组件标准

### 8.1 首页主控制卡

作为全产品主控制类卡片的模板：

- 主标题 + 状态徽标 + 简短说明 + 主开关
- 内容内边距：`28`
- 圆角：`24`
- 可使用轻材质
- 状态蓝徽标采用低对比填充，不做艳丽色块

### 8.2 常规设置卡

作为参数控制和配置块的模板：

- 标题
- 辅助说明
- 当前值或摘要
- 控件本体
- 可选脚注

标准：

- 内边距：`24`
- 圆角：`22`
- 背景为深色实体面
- 用极轻蓝色描边或低对比覆盖层提示产品主色

### 8.3 摘要条

摘要条用于：

- 首页低优先级统计
- 规则页顶部简报
- 日志页顶部总量/状态简报

标准：

- 只能作为低优先级信息存在
- 不允许再次演变成四张彩色统计卡片
- 默认单行或紧凑多段式表达

### 8.4 表单控件

- Slider、Toggle、选中态继续使用系统默认行为
- 强调色统一为蓝色
- 不额外包装成网页式“自定义控件”

### 8.5 列表与日志

规则页和日志页未来改造时采用以下原则：

- 列表区必须落在稳定深色表面上
- 搜索/过滤条如果需要，做成低对比工具条，不做大面积玻璃
- 行分隔、hover、选中统一走克制路线
- 日志页的颜色只服务于日志级别语义，不用于页面装饰

## 9. 色彩标准

### 9.1 主色

- 主强调色：系统蓝 / 产品蓝
- 使用场景：
  - 左侧导航选中行
  - 首页状态徽标
  - Slider 已激活轨道
  - 关键数值
  - 可交互焦点

### 9.2 语义色

- 绿色：仅用于“启用/健康/成功”点状或小型状态
- 黄色/橙色：仅用于警告
- 红色：仅用于错误、危险操作、不可逆行为

### 9.3 禁止事项

- 禁止为“丰富度”引入额外主题色
- 禁止把紫色、青色、绿色当作平级品牌强调色使用
- 禁止大面积彩色填充卡片

## 10. 圆角、描边与阴影标准

### 10.1 圆角层级

推荐语义层级：

- 小型图标底板：`8`
- 品牌图标容器：`10`
- 选中导航行：`12`
- 常规功能卡：`22`
- 主控制卡：`24`

如果后续新增组件，优先复用这组层级，不新增杂乱圆角值。

### 10.2 描边

- 描边必须轻
- 主要用于分层，而不是画轮廓
- 右侧卡片的描边默认可带轻蓝色倾向
- 左侧侧栏分隔使用中性浅线

### 10.3 阴影

- 阴影只能作为辅助，不是主层级工具
- 左侧栏可保留极轻横向层次阴影
- 右侧卡片默认不依赖重阴影

## 11. 间距标准

### 11.1 页面级

- 页面整体 padding：`30`
- 顶级模块间距：`24`

### 11.2 卡片级

- Hero 卡片 padding：`28`
- 常规卡片 padding：`24`

### 11.3 列表级

- 导航行高：`46`
- 导航项间距：`6`
- 导航区顶部额外呼吸：`8`

后续页面可在密度上微调，但必须保持“页面级 > 卡片级 > 行级”的稳定节奏。

## 12. 明确禁止的视觉模式

后续页面改造时，默认禁止以下做法：

- 重新出现网页式大蓝色按钮导航
- 重新出现多彩统计卡阵列
- 右侧内容区整页玻璃化
- 大面积渐变装饰背景
- 夸张顶部图标块或 hero 图标卡
- 用更多颜色而不是更清晰结构解决层级问题

## 13. 页面扩展规则

### 13.1 应用规则页

未来改造时应遵循：

- 右侧保持深色实体工具面
- 规则列表/搜索/过滤区作为高密度工具界面处理
- 主色仅用于当前筛选、焦点、选中和关键规则状态
- 不把规则页做成营销式设置中心

### 13.2 日志页

未来改造时应遵循：

- 更偏向专业工具面板
- 重点在可读性、密度和过滤效率
- 背景稳，层级清楚，不引入玻璃秀场

### 13.3 新增页面

任何后续新增页面都必须先回答两件事：

- 它属于“主控制页”还是“工具列表页”
- 它应该继承左栏/右侧这套分工中的哪一类表面

如果回答不清，就不应该直接开始做 UI。

## 14. 实现建议

这份规范的下一步工程动作应当是：

- 在后续页面改造前，把已稳定的尺寸、圆角、描边和颜色抽成共享 token
- 把主控制卡、常规卡、状态徽标、侧栏导航行等抽成可复用视图
- 后续每个页面先对照本规范判断属于哪类页面，再实施局部改造

## 15. 验收标准

后续所有页面如果宣称“符合 HiWindowGuy 新风格”，至少要满足以下条件：

- 第一眼像 macOS 工具，而不是网页后台
- 左栏与右侧内容各司其职，没有互相抢主视觉
- 蓝色是唯一主强调色，其他颜色只服务语义
- 玻璃感只出现在允许的结构层，不泛滥
- 卡片、列表、标题、状态表达符合统一节奏
- 新页面接进来不会像另一套产品

## 16. 规范优先级

从今天起，这份文档优先级高于此前单页 spec 中的局部表达。

适用方式如下：

- 首页继续以 `2026-04-15-homepage-control-panel-design.md` 作为页面结构补充
- 左侧栏继续以 `2026-04-17-sidebar-apple-cues-design.md` 作为局部实现补充
- 所有后续页面改造，以本规范作为最高层视觉与组件标准

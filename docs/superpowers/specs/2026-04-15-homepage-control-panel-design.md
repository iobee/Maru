# HiWindowGuy 首页重设计 Spec

日期：2026-04-15  
状态：已完成设计确认，待实现  
范围：仅重做首页，不包含“应用规则”和“日志”页面

## 1. 目标

把当前首页从“网页风卡片面板”调整为更成熟的 macOS 工具首页，让它看起来像一个稳定、专业的控制面板，而不是一个还没收住风格的后台页面。

本次设计的核心目标：

- 提升专业感和完成度
- 强化首页的控制属性，而不是信息展示属性
- 引入轻度玻璃感，但保持克制
- 在 macOS 13+ 下继续使用现有 SwiftUI 能力实现兼容方案

## 2. 用户已确认的设计决策

- 视觉方向：`B. Refined Utility`
- 材质方向：`轻度玻璃`
- 首轮改造范围：`仅首页`
- 首页定位：`主控制面板`
- 统计信息处理方式：`保留，但缩成低优先级摘要`
- 首页布局方案：`A. Hero Control`

## 3. 问题诊断

当前首页的“业余感”主要来自以下几点：

- 页面层级主要依赖更亮的蓝色、更粗的字重和更多卡片，而不是稳定的信息结构
- 每张卡片的材质、阴影、圆角语言太接近，页面像堆叠的容器集合
- 侧边栏过宽，首页内容又过散，视觉重心不稳
- 首页顶部图标、功能卡片、统计卡片使用了多套不统一的视觉语言
- 彩色统计卡片过于抢眼，夺走了首页作为控制面板的主角色

## 4. 设计原则

新版首页遵循以下原则：

- 控制优先：用户进入首页后，应该先看到状态与主操作
- 克制材质：玻璃感只用于结构层和主控制区，不做满屏磨砂
- 单一强调色：尽量使用同一套蓝色强调体系，不靠多色制造“丰富”
- 层级靠节奏：优先通过留白、间距、字重、明度差建立层级
- macOS 工具感：更像原生桌面工具，而不是网页仪表盘

## 5. 视觉语言

### 5.1 材质策略

- 侧边栏、首页主控制卡、顶部结构区可使用轻度玻璃材质
- 次级控制区使用更稳的半实色表面
- 避免每个卡片都使用 `Material`，否则页面会重新变得松散和廉价
- 不追求系统新版本专属“Liquid Glass”特效 API，而是使用现有 SwiftUI `Material`、细描边、顶部高光和分层背景去复现相近气质

### 5.2 色彩策略

- 保留蓝色作为唯一主强调色
- 统计区不再以紫、绿、灰等多色块制造存在感
- 非重点信息更多使用明度和透明度区分

### 5.3 排版策略

- 页面标题、副标题、模块标题三层足够，不再叠加过多粗体和大字号
- 主控制卡内的状态、说明和开关形成稳定左右结构
- 首页整体增加留白，让主控制区真正成为视觉锚点

## 6. 首页最终信息结构

首页只保留以下 4 个区域，顺序固定：

### 6.1 页面头部

内容：

- 页面标题：窗口管理
- 一句简洁副标题：管理和控制窗口尺寸和位置

调整要求：

- 移除当前首页顶部那种强装饰性的蓝色大图标块
- 标题区存在感降低，为主控制区让位

### 6.2 主控制卡

这是首页第一视觉锚点。

承载内容：

- 当前状态：已启用 / 已停用
- 功能说明：自动调整窗口大小和位置
- 主开关
- 可选辅助信息：例如“当前正在监听前台应用”

视觉要求：

- 使用轻度玻璃感表面
- 带细描边和轻微顶部高光
- 开关成为最强交互焦点
- 文字说明不宜过多

### 6.3 缩放控制卡

这是首页第二重点区域，用来承接参数设置。

承载内容：

- 标题：窗口缩放比例
- 说明：控制「几乎最大化」时窗口的大小
- 当前比例数值
- Slider
- 辅助标签：更紧凑 / 更宽敞

视觉要求：

- 使用半实色表面，而不是强玻璃材质
- 保持与主控制卡一致的圆角、描边和留白节奏
- 当前百分比要明确，但不能喧宾夺主

### 6.4 统计摘要条

这是首页最低优先级区域，只作为补充信息存在。

建议形式：

`已记录 130 个应用 · 居中 11 · 几乎最大化 112 · 忽略 7`

视觉要求：

- 使用一条低对比摘要条代替 4 张彩色统计卡片
- 摘要可采用单行或四段式紧凑布局
- 视觉权重低于主控制卡和缩放控制卡

## 7. 需要删除或弱化的内容

- 删除首页顶部的蓝色图标块式标题表现
- 删除当前 4 张彩色统计卡片
- 弱化侧边栏当前过重的选中态表现
- 减少多余阴影，改用材质差、描边和亮度差分层
- 避免页面中每个模块都看起来像独立的网页卡片组件

## 8. 交互与状态要求

- 开关状态切换后，主控制卡中的状态文案需要同步变化
- 缩放 Slider 的当前百分比显示保持实时更新
- 统计摘要仍读取现有 `AppConfig.appRules` 数据，不新增数据来源
- 主页不新增复杂交互，不引入新的页面级功能

## 9. 实现边界

本次只允许调整首页的视觉结构和表现，不包含：

- `RuleConfigView` 重设计
- `LogViewer` 重设计
- 新增首页快捷操作
- 新增图表、动效系统或复杂图标系统
- 提高最低 macOS 版本或依赖仅新系统可用的专属 UI API

## 10. 验收标准

完成后应满足以下条件：

- 首页第一眼更像控制面板，而不是卡片仪表盘
- 用户能在 2 秒内理解当前是否启用，以及主要可调参数是什么
- 主控制卡、缩放控制卡、摘要条之间存在明确主次
- 页面视觉语言统一，不再依赖多色统计卡片制造丰富度
- 在深色模式下保持干净、稳定，不出现过度发蓝或材质泛滥

但不要求在本轮中系统性改造其他页面。

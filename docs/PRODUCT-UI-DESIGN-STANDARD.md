# Maru 产品 UI 设计规范

状态：正式生效  
最近整理：2026-04-18  
适用范围：主窗口内所有现有页面与后续新增页面  
不适用范围：菜单栏图标、系统权限弹窗、系统原生对话框

## 1. 文档定位

本文件是 Maru 主窗口 UI 的正式设计规范，用于统一首页、应用规则页、日志页以及未来新增页面的视觉方向、布局规则、组件样式和验收标准。

这份规范回答的是“Maru 应该长期长成什么样”，而不是单个页面某一轮改版时的临时想法。后续所有 UI 改动都应先对齐本文件，再进入实现。

## 2. 优先级与引用关系

本文件是主窗口 UI 的最高优先级设计依据。

优先级顺序如下：

1. 本文件：`docs/PRODUCT-UI-DESIGN-STANDARD.md`
2. 页面级补充规范：
   - `docs/superpowers/specs/2026-04-15-homepage-control-panel-design.md`
   - `docs/superpowers/specs/2026-04-17-sidebar-apple-cues-design.md`
3. 具体实现文件中的局部样式细节

如果页面级 spec 与本文件冲突，以本文件为准。页面级 spec 只负责补充局部页面结构，不得推翻产品级标准。

## 3. 产品设计定义

### 3.1 产品气质

- 产品定位：`macOS utility console`
- 品牌主张：`Center it beautifully.`
- 中文主张：`一键居中，让日常更优雅。`
- 设计目标：像原生 macOS 工具，而不是网页后台或营销面板
- 首页定位：主控制面板，不是 dashboard

### 3.2 一句话风格定义

Maru 的标准风格是：

`左侧栏保留轻度原生玻璃感，右侧内容区以稳定深色实体表面为主，信息层级依赖结构、描边和留白，而不是依赖多色块、重阴影和夸张材质。`

### 3.3 关键决策

- 左侧栏方向：`Apple Cues Only`
- 材质策略：`restrained sidebar glass + solid dark content surfaces`
- 主强调色：蓝色
- 玻璃策略：轻度玻璃，只用于结构层和少量主控制层
- 搜索框：默认不引入，除非页面任务模型明确需要

## 4. 核心设计原则

### 4.1 结构先于装饰

- 专业感来自信息结构、块级关系和交互优先级
- 不依赖彩色卡片、粗阴影、大面积发光制造“设计感”

### 4.2 左右分工明确

- 左侧栏负责导航、产品识别和轻度氛围
- 右侧内容区负责功能表达和操作承载
- 左右两侧不得争抢主视觉中心

### 4.3 材质必须克制

- 玻璃感只允许用于结构层和少量主控制层
- 不允许整页多个区域同时大面积使用强材质
- 默认优先稳定实体表面，其次才是材质氛围

### 4.4 蓝色是唯一主强调色

- 蓝色用于选中态、主开关、关键数值、焦点和主状态徽标
- 绿色、黄色、红色仅用于语义表达，不参与装饰
- 不再引入紫色、青色、绿色等平级品牌强调色

### 4.5 桌面工具优先，不做网页后台

- 避免网页式统计卡阵列
- 避免大块 CTA 感选中项
- 避免满屏圆角卡片彼此竞争

## 5. 页面结构模型

所有主窗口页面都遵循“左侧结构层 + 右侧内容层”的双区模型。

### 5.1 左侧结构层

用途：

- 产品品牌识别
- 一级导航
- 底部运行状态

约束：

- 不承担主内容的视觉中心
- 不承载复杂业务交互
- 不承载大块统计信息

### 5.2 右侧内容层

用途：

- 页面标题与说明
- 主功能卡片
- 参数控制
- 列表、日志、规则编辑等业务内容

约束：

- 主功能卡片优先位于页面上半部
- 页面内只保留清晰的一级、二级视觉重点
- 次级信息用较低对比和较小字号处理，不抢主内容

## 6. 设计 Token

### 6.1 布局 Token

| 项目 | 值 |
| --- | --- |
| 侧栏宽度 | `248` |
| 页面水平内边距 | `34` |
| 页面垂直内边距 | `30` |
| 顶级模块间距 | `24` |
| 侧栏顶部内边距 | `20` |
| 侧栏水平内边距 | `14` |
| 侧栏底部内边距 | `14` |
| 导航行高 | `46` |
| 导航项间距 | `6` |
| 导航区顶部额外留白 | `8` |

说明：

- 个别页面可按信息密度做轻微调整
- 不得破坏“页面级 > 卡片级 > 行级”的整体节奏

### 6.2 卡片与组件 Token

| 项目 | 值 |
| --- | --- |
| 主控制卡内边距 | `28` |
| 常规卡内边距 | `24` |
| 主控制卡圆角 | `24` |
| 常规卡圆角 | `22` |
| 选中导航行圆角 | `12` |
| 品牌图标容器圆角 | `10` |
| 小型图标底板圆角 | `8` |
| 品牌图标尺寸 | `38 x 38` |

### 6.3 色彩 Token

| 类别 | 规则 |
| --- | --- |
| 主强调色 | 系统蓝 / 产品蓝 |
| 成功色 | 仅用于启用、健康、成功语义 |
| 警告色 | 仅用于警告语义 |
| 危险色 | 仅用于错误、危险操作、不可逆行为 |

禁止事项：

- 不得为了“丰富度”增加主题色
- 不得使用大面积彩色填充卡片
- 不得把语义色当作第二套品牌色使用

## 7. 左侧栏标准

左侧栏是全产品唯一允许长期保留原生玻璃感的结构区。

### 7.1 材质

- 使用原生 `NSVisualEffectView` 侧栏材质桥接
- 默认材质：`.sidebar`
- 当前混合方式：`behindWindow`
- 不额外叠加顶部高光
- 可保留极轻分隔线和轻阴影，但不能叠加厚重发光

### 7.2 容器

- 使用贴边面板，不做悬浮卡片式侧栏
- 背景延伸到标题栏后方，使 traffic lights 处于同一材质之内
- 右侧只保留一条轻分隔，不做厚描边容器

### 7.3 品牌区

- 保留 `Maru + 图标`
- 品牌区表达产品名，不表达当前页面标题
- 品牌区保持紧凑，不使用横幅式大标题布局

### 7.4 导航项

- 导航项表现为列表中的一行，不是网页按钮
- 当前选中项必须是明显蓝色高亮行
- 图标底板可以保留，但必须统一尺寸与圆角
- 默认不增加搜索、不增加账户卡、不增加复杂分组

## 8. 右侧内容区标准

右侧内容区默认使用稳定、克制的深色实体表面，不允许整体变成“第二块玻璃面板”。

### 8.1 页面容器

- 默认不设固定内容宽度
- 通过统一页边距控制留白，不制造网页式单边空白
- 如果某页因信息密度需要特殊宽度限制，必须作为明确例外说明

### 8.2 文本层级

- 页面标题：`largeTitle + bold`
- 页面副标题：`body + secondary`
- 模块标题：`title2 / title3 + semibold`

每页最多维持三层主要文字层级：

- 页面标题
- 模块标题
- 正文 / 说明

不得继续叠加更多粗体和更大的字号制造层级。

### 8.3 内容表面分工

#### A. 主控制表面

适用场景：

- 首页 Hero Control
- 页面主开关区域
- 承载最高优先级交互的卡片

规则：

- 可使用轻度材质或极浅玻璃感
- 必须配合细描边
- 视觉上保持克制，不能做成漂浮果冻块

#### B. 常规功能表面

适用场景：

- 参数控制卡
- 规则编辑块
- 日志过滤区与列表承载区
- 辅助信息模块

规则：

- 以深色实体面为主
- 用轻描边、轻色偏和圆角建立层次
- 不使用强玻璃

## 9. 组件标准

### 9.1 主控制卡

标准结构：

- 主标题
- 状态徽标
- 简短说明
- 主开关

样式规则：

- 内边距 `28`
- 圆角 `24`
- 允许轻材质
- 蓝色状态徽标采用低对比填充，不做高饱和色块

### 9.2 常规设置卡

标准结构：

- 标题
- 辅助说明
- 当前值或摘要
- 控件本体
- 可选脚注

样式规则：

- 内边距 `24`
- 圆角 `22`
- 背景为深色实体面
- 可使用极轻蓝色描边或低对比覆盖层表达主色

### 9.3 摘要条

适用场景：

- 首页低优先级统计
- 规则页顶部简报
- 日志页顶部总量或状态简报

规则：

- 只能作为低优先级信息存在
- 不允许演变成四张彩色统计卡片
- 默认采用单行或紧凑多段式表达

### 9.4 表单控件

- `Slider`、`Toggle`、选中态优先保持系统默认行为
- 强调色统一使用蓝色
- 不额外包装为网页式自定义控件

### 9.5 列表与日志

- 列表区必须落在稳定深色表面上
- 搜索或过滤条如有需要，应做成低对比工具条
- 行分隔、hover、选中统一走克制路线
- 日志颜色只服务于日志级别语义，不用于页面装饰

## 10. 圆角、描边与阴影

### 10.1 圆角

优先复用以下层级，不新增杂乱圆角值：

- 小型图标底板：`8`
- 品牌图标容器：`10`
- 选中导航行：`12`
- 常规功能卡：`22`
- 主控制卡：`24`

### 10.2 描边

- 描边必须轻
- 描边用于分层，不用于画轮廓
- 右侧卡片描边可带极轻蓝色倾向
- 左侧栏分隔使用中性浅线

### 10.3 阴影

- 阴影只能作为辅助，不是主要层级工具
- 左侧栏可保留极轻横向层次阴影
- 右侧卡片默认不依赖重阴影

## 11. 明确禁止的视觉模式

以下模式默认禁止出现在 Maru 主窗口 UI 中：

- 网页式大蓝色按钮导航
- 多彩统计卡阵列
- 右侧内容区整页玻璃化
- 大面积渐变装饰背景
- 夸张顶部图标块或 hero 图标卡
- 用更多颜色替代更清晰的结构

## 12. 页面适配规则

### 12.1 首页

- 作为主控制页处理
- 保持“主控制卡 + 低优先级摘要 + 辅助控制块”的层级
- 不回退到网页式 dashboard 模式

### 12.2 应用规则页

- 作为高密度工具页处理
- 规则列表、搜索、过滤优先保证效率和可读性
- 主色只用于当前筛选、焦点、选中和关键规则状态
- 不做营销式设置中心

### 12.3 日志页

- 更偏向专业工具面板
- 重点在密度、可读性和过滤效率
- 背景稳、层级清楚，不做玻璃秀场

### 12.4 新增页面

设计前必须先回答两件事：

1. 它是“主控制页”还是“工具列表页”？
2. 它应该继承哪一类右侧内容表面？

如果这两个问题回答不清，不应直接开始做 UI。

## 13. 实现要求

后续实现应优先做以下事情：

- 抽取共享尺寸、圆角、描边和颜色 token
- 抽取可复用视图：主控制卡、常规卡、状态徽标、侧栏导航行
- 每次新页面改造前，先判断页面类型，再决定组件和材质策略

## 14. 验收标准

一个页面只有同时满足以下条件，才可以宣称“符合 Maru 产品 UI 标准”：

- 第一眼像 macOS 工具，而不是网页后台
- 左栏与右侧内容各司其职，没有互相抢主视觉
- 蓝色是唯一主强调色，其他颜色只服务语义
- 玻璃感只出现在允许的结构层，不泛滥
- 卡片、列表、标题、状态表达符合统一节奏
- 新页面接入后不会像另一套产品
